🌐 基础 HTTP / Curl 极简上报 API 单条 API 命令 ▾

支持使用任意语言或工具发起 HTTP POST 请求进行单个解析线路的 IP 强制刷新。

Standard Shell Curl 指令：
curl -X POST "http://[本站域名或IP]/api/ddns/update" \ -H "Authorization: Bearer [对应线路的Token]" \ -d "ip=1.1.1.1"
🐧 Linux VPS 服务器 (Ubuntu / Debian / CentOS) - Cloudflare 三网优选自动同步整套 Shell 脚本 推荐 VPS 使用 ▾

本脚本会自动依赖检查与安装（兼容 apt-get 与 yum），自动下载 CloudflareSpeedTest 工具进行测速，提取测出的电信 (CT)、联通 (CU)、移动 (CM) 前1名最佳 IP 并自动批量提交推送到 TriNet DNS。

开箱即用三网自动测速更新 Shell 脚本 (cf_speedtest_trinet.sh)：
#!/bin/bash # ========================================================= # TriNet DNS - Cloudflare 三网优选 IP 自动测速与同步上报脚本 # 适配系统: Ubuntu / Debian / CentOS / Alpine / RHEL # ========================================================= # --- 【请在此配置您在 TriNet DNS 后台生成的 4 个 Token 与服务地址】 --- TRINET_SERVER="http://[您的TriNet服务器域名或IP]" TOKEN_DEF="[填写默认线路Token]" # 默认/三网通用 (DEF) TOKEN_CT="[填写电信线路Token]" # 中国电信 (CT) TOKEN_CU="[填写联通线路Token]" # 中国联通 (CU) TOKEN_CM="[填写移动线路Token]" # 中国移动 (CM) # ------------------------------------------------------------- # 1. 自动依赖校验 if command -v apt-get &> /dev/null; then command -v curl &> /dev/null || apt-get update && apt-get install -y curl tar wget elif command -v yum &> /dev/null; then command -v curl &> /dev/null || yum install -y curl tar wget fi WORKDIR="/tmp/cf_speedtest_trinet" mkdir -p "$WORKDIR" && cd "$WORKDIR" # 2. 下载并解压 CloudflareSpeedTest 最新工具 (若本地未下载) if [ ! -f "./CloudflareST" ]; then echo "[INFO] 正在获取 CloudflareSpeedTest 测速引擎..." wget -N --no-check-certificate https://github.com/XIU2/CloudflareSpeedTest/releases/latest/download/CloudflareST_linux_amd64.tar.gz -O CloudflareST.tar.gz tar -zxf CloudflareST.tar.gz chmod +x CloudflareST fi # 3. 运行测速 (输出结果至 result.csv) echo "[INFO] 正在开始三网节点网络延迟与下载速度测试..." ./CloudflareST -o result.csv -n 200 -dn 3 if [ ! -f "result.csv" ]; then echo "[ERROR] 测速失败，未生成 result.csv 文件" exit 1 fi # 4. 提取优选前 3 个响应最佳的 IP BEST_IP_1=$(sed -n '2p' result.csv | awk -F, '{print $1}') BEST_IP_2=$(sed -n '3p' result.csv | awk -F, '{print $1}') BEST_IP_3=$(sed -n '4p' result.csv | awk -F, '{print $1}') [ -z "$BEST_IP_1" ] && BEST_IP_1="104.16.1.1" [ -z "$BEST_IP_2" ] && BEST_IP_2="$BEST_IP_1" [ -z "$BEST_IP_3" ] && BEST_IP_3="$BEST_IP_1" echo "[INFO] 测速完成！选出三网最佳 IP: 默认/电信->$BEST_IP_1, 联通->$BEST_IP_2, 移动->$BEST_IP_3" # 5. 调用 TriNet DNS 接口进行同步更新 [ -n "$TOKEN_DEF" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_DEF" -d "ip=$BEST_IP_1" && echo " [默认] 已更新" [ -n "$TOKEN_CT" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_CT" -d "ip=$BEST_IP_1" && echo " [电信] 已更新" [ -n "$TOKEN_CU" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_CU" -d "ip=$BEST_IP_2" && echo " [联通] 已更新" [ -n "$TOKEN_CM" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_CM" -d "ip=$BEST_IP_3" && echo " [移动] 已更新" echo "[SUCCESS] 全线路智能解析同步完毕！"
💡 定时任务设置提示 (Linux Crontab)：在终端运行 crontab -e 追加一行：
*/30 * * * * bash /path/to/cf_speedtest_trinet.sh &> /dev/null（即每 30 分钟自动测试并刷新三网解析）。
🌐 方案二：无需本地测速，直接从开源 API 自动获取最新三网优选 IP 脚本 (fetch_online_cf_ip.sh)：
#!/bin/bash # ========================================================= # TriNet DNS - 自动抓取开源 API 三网优选 IP 并推送上报脚本 # 无需本地测速环境，自动提取网络最新测试的三网极速节点 IP # ========================================================= TRINET_SERVER="http://[您的TriNet服务器域名或IP]" TOKEN_DEF="[填写默认线路Token]" # 默认/三网通用 (DEF) TOKEN_CT="[填写电信线路Token]" # 中国电信 (CT) TOKEN_CU="[填写联通线路Token]" # 中国联通 (CU) TOKEN_CM="[填写移动线路Token]" # 中国移动 (CM) echo "[INFO] 正在调用专业 API 抓取全网最新 默认+三网 优选 IP..." API_DATA=$(curl -s "https://jkapi.com/api/cf_best?server=1&type=v4") # 从 jkapi 接口精准提取默认/三网通用(CN)、电信(CT)、联通(CU)、移动(CM) 最佳 IP DEF_IP=$(echo "$API_DATA" | grep -oE '"CN":\[[^]]*' | grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | head -n 1) CT_IP=$(echo "$API_DATA" | grep -oE '"CT":\[[^]]*' | grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | head -n 1) CU_IP=$(echo "$API_DATA" | grep -oE '"CU":\[[^]]*' | grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | head -n 1) CM_IP=$(echo "$API_DATA" | grep -oE '"CM":\[[^]]*' | grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | head -n 1) # 如果主 API 超时，自动回退备用源 if [ -z "$CT_IP" ] || [ -z "$CU_IP" ] || [ -z "$CM_IP" ]; then echo "[WARN] 主源未响应，切换备用源抓取..." IPS=$(curl -s https://ip.164746.xyz/ | grep -oE '[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+' | sort -u) [ -z "$DEF_IP" ] && DEF_IP=$(echo "$IPS" | sed -n '1p') [ -z "$CT_IP" ] && CT_IP=$(echo "$IPS" | sed -n '1p') [ -z "$CU_IP" ] && CU_IP=$(echo "$IPS" | sed -n '2p') [ -z "$CM_IP" ] && CM_IP=$(echo "$IPS" | sed -n '3p') fi [ -z "$DEF_IP" ] && DEF_IP="104.16.1.1" [ -z "$CT_IP" ] && CT_IP="104.16.1.1" [ -z "$CU_IP" ] && CU_IP="104.17.2.2" [ -z "$CM_IP" ] && CM_IP="104.18.3.3" echo "[SUCCESS] 抓取完成！" echo " - 🌐 默认/三网通用 最佳 IP: $DEF_IP" echo " - ⚡ 中国电信 (CT) 最佳 IP: $CT_IP" echo " - ⚡ 中国联通 (CU) 最佳 IP: $CU_IP" echo " - ⚡ 中国移动 (CM) 最佳 IP: $CM_IP" [ -n "$TOKEN_DEF" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_DEF" -d "ip=$DEF_IP" && echo " ✅ 默认线路解析记录已更新" [ -n "$TOKEN_CT" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_CT" -d "ip=$CT_IP" && echo " ✅ 电信解析记录已更新" [ -n "$TOKEN_CU" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_CU" -d "ip=$CU_IP" && echo " ✅ 联通解析记录已更新" [ -n "$TOKEN_CM" ] && curl -s -X POST "$TRINET_SERVER/api/ddns/update" -H "Authorization: Bearer $TOKEN_CM" -d "ip=$CM_IP" && echo " ✅ 移动解析记录已更新" echo "[ALL DONE] 全线路解析记录已成功推送到面板！"
📶 OpenWrt 软路由 - 动态公网 IP 自动上报脚本与计划任务 路由器专属 ▾

适用于 OpenWrt 路由器环境，自动检测 WAN 口公网 IP 变动并向 TriNet DNS 进行同步（适配 curl / wget-ssl）。

OpenWrt 上报脚本 (trinet_openwrt_ddns.sh)：
#!/bin/sh # OpenWrt DDNS 上报脚本 TRINET_SERVER="http://[您的TriNet服务器域名或IP]" TOKEN="[填写您的DDNS_Token]" # 提取 WAN 口公网 IP WAN_IP=$(curl -s http://api.ipify.org || wget -qO- http://api.ipify.org) if [ -n "$WAN_IP" ]; then curl -s -X POST "$TRINET_SERVER/api/ddns/update" \ -H "Authorization: Bearer $TOKEN" \ -d "ip=$WAN_IP" fi
💡 OpenWrt 定时任务部署：登录 OpenWrt Web 后台 -> 【系统】 -> 【计划任务】，加入：
*/5 * * * * sh /etc/trinet_openwrt_ddns.sh &> /dev/null（每 5 分钟自动上报最新公网 IP）。
⚡ MikroTik RouterOS (ROS) 路由器 - 原生 Fetch 上报脚本 ROS 专属 ▾

使用 RouterOS 原生脚本语言和 /tool fetch 命令向 TriNet DNS 进行更新。

RouterOS (ROS) 脚本命令：
# ROS RouterOS Script :local trinetServer "http://[您的TriNet服务器域名或IP]/api/ddns/update" :local token "[填写您的DDNS_Token]" # 获取 WAN 口 IP :local wanInterface "pppoe-out1" :local currentIP [/ip address get [/ip address find interface=$wanInterface] address] :set currentIP [:pick $currentIP 0 [:find $currentIP "/"]] :if ([:len $currentIP] > 0) do={ /tool fetch url=$trinetServer http-method=post http-header-field="Authorization: Bearer $token" http-data=("ip=" . $currentIP) keep-result=no :log info ("TriNet DDNS Updated: " . $currentIP) }
💡 ROS 定时器部署：在 WinBox Terminal 输入命令配置 5 分钟定时器：
/system scheduler add name=trinet-ddns interval=5m on-event="/system script run trinet-script"
